# Spec Gap Audit F — 實作層 placeholder / no-op 缺口
Date: 2026-05-06
Scope: Pack E E0–E8 落地後，使用者實機驗收發現的「UI 存在但行為缺失」缺口
Status: **OPEN — 待 Pack D / 規劃團隊 disposition**
Owner: implementation-side audit（spec 是否要求修由規劃端裁示）

## 背景

Pack E 完成後使用者實際點按 `/management/personas`、`/management/capital`、`/management/strategies` 等 list 頁面右上角的「建立」按鈕，發現按下無反應。回查程式碼確認此**非 Pack E 退化**，而是 v3/v4 期間 `ObjectListPage` 第 40 行就一直沒接 `onClick` 的歷史 placeholder。同檔還帶了一個 pre-Pack-E 的 React `forwardRef` warning。

Pack E disposition 28 條未覆蓋此範圍。本份 audit 把實作層缺口顯式記錄，避免後續 smoke 報告再次誤標「regression-clean」。

## 推測的 Pack 歸屬

- **F01 / F03**：屬 Pack D 或後續 Pack F 規範範圍（CRUD shape + IA 收斂）。不在 Pack E disposition 28 條內。F01 與 Audit D blocker D05（async transition timeout/failureState）在語意上同源 —— 都是「write contract 未定」的不同切面。
- **F02**：純實作 bug，不需 spec 即可修，但建議與 F01 一起排程。
- **F04**：僅 audit-trail，無需 spec。

## 建議下一步

1. 規劃團隊對 F01 / F03 給出 disposition：
   - 是否在 v0-mock 層補 Create / Edit Drawer（寫入 `v5ActionOverlay` 或直接寫入 mock seed）？
   - 或維持唯讀直到真實後端就緒，並把「建立」按鈕標 `disabled + tooltip "待後端啟用"`？
2. F02 由實作端在收到 F01 disposition 時順手修掉（forwardRef 包一層）。
3. **在收到 disposition 之前，實作端不主動在 `src/` 新增任何 Create flow**，避免重蹈 Audit D「實作搶在 spec 前面」教訓。
