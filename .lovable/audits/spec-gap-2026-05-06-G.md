# Spec Gap Audit G — Pack F 落地後第三輪盤點

- 日期：2026-05-06
- 範圍：Pack F（Write Intent + IA Boundary + QA Semantics）落地後浮現的 second-order gap
- 來源檔案：`src/lib/writeIntents/`、`src/lib/bff/writeOverlay.ts`、`src/management/components/write/EntityCreateDrawer.tsx`、`src/management/pages/v5/ExecutionLoop.tsx`、`src/management/pages/v5/OptimizationLoop.tsx`、`src/management/pages/studios/FormulaStudio.tsx`、`src/management/pages/Lists.tsx`
- 收斂狀態：**OPEN — 待 Pack G disposition 或併入 Pack D 範圍**

---

## 條目

### G01 — High — WriteIntent vs Pack D D05 timeout 對齊未定
v0 mock create 立即成功；但真實後端進入 async transition 時，create→draft→ready 的 SLA / failureState（D05）尚未明訂於 writeIntents 規範。`CreateIntentResult` 缺 `pendingTransitionId` / `expectedReadyBy` 欄位。
**建議**：併入 Pack D D05 規範範圍，補 `CreateIntentResult.pendingTransition?: { id, timeoutMs, failureState }`。

### G02 — High — `createBehavior=redirect` 的 intent contract 未定
`Lists.tsx` 對 ranking/rebalance 走 `navigate(to, { state: { intent: "create" } })`，但目標頁（FormulaStudio / OptimizationLoop）對 `location.state.intent` 的接收契約未在 spec 定義。第三方 deep-link 無法重現。
**建議**：規範統一改用 query string `?intent=create` 並列出每個目標頁應有的 onMount 行為（open dialog / focus field / scroll）。

### G03 — High — `EntityCreateDrawer` 9 entity 共用 form，但 spec §2.5 要求 per-entity 進階欄位
目前所有 entity 共用 `<Input>` / `<Textarea>`，缺：
- `currency` / `target` / `kind` / `initialMode` 應為 `<Select>`（受控 enum）
- `personaIds` 應為 multi-select（目前是 comma-separated string）
- `riskBudget` 應有 slider
**建議**：spec 補 per-entity field component spec；impl 拆 `EntityCreateDrawer` 子元件。

### G04 — High — Drawer 無 permission guard
`EntityCreateDrawer` 未透過 `usePermissions().can("entity.create")` 守門。任何 role 都能 create，違反 v4 `permissionsMatrix`。
**建議**：每個 `CreatableEntity` 對應 `entity.create` capability；ObjectListPage 在 render Create 按鈕時即過濾。

### G05 — Medium — writeOverlay 30min TTL 與 audit log append 不對稱
`writeOverlay.add` 30min 後 GC，但同時 push 到 `auditEvents.unshift(...)`（mock seed 持久）。重整後 list 看不到項目，但 audit log 看得到 `entity.create`，造成「孤兒審計事件」。
**建議**：要嘛 audit 也走 overlay，要嘛 overlay 不過期（或延長至 session 結束）。

### G06 — Medium — `ENTITY_TO_LIVE_KIND` 字串映射未對應 SSE D26 channel schema
寫死字串（`"Strategy"`, `"Persona"` …）與 v5 `bff.realtime` channel 對焊未驗證。Pack D D26 規範未到，但本 mapping 應與 channel 命名綁定。
**建議**：等 D26 落地後改為從 `SSE_CHANNELS` 常數導出。

### G07 — Medium — `/loops/execution` focus 對稱性不全
ExecutionLoop 接 `?focus=personas|strategies|deployments`，但只實作 personas/strategies 對應 scroll target；strategies / deployments 都 scroll 到同一 `runsRef`。OptimizationLoop **完全無 focus 機制**（rebalance 入口缺）。
**建議**：定義 `focus` enum per loop，列出每個 focus 對應的 scroll anchor / panel filter / breadcrumb 標籤。

### G08 — Medium — IA boundary 文案在三層（SideNav label / PageHeader title / Breadcrumb）未統一
目前 `/management/personas` SideNav 顯示 "Persona Registry"，但 PersonasList PageHeader 仍是 "Personas"，breadcrumb 無 registry 字樣。
**建議**：規範三層文案來源（單一 i18n key）。

### G09 — Medium — Drawer submit 無 idempotency token / optimistic-lock 欄位
v4 `idempotency.ts` / `optimisticLock.ts` 已存在，但 `EntityCreateDrawer.submit` 未產生 `Idempotency-Key`，未來接後端會雙寫。
**建議**：spec 列 create 流程的 idempotency 來源（drawer 開啟時 mint key）。

### G10 — Low — `EntityCreateDrawer` field labels 未走 i18n
`ENTITY_LABELS`、`FieldDef.label` 全為英文 hardcode。違反 i18n 全站走 key 的規則。
**建議**：補 `entityCreate.{entity}.title`、`entityCreate.fields.{name}` keys，至少 zh-TW + en-US。

### G11 — Low — `EntityCreateDrawer` 無 a11y aria-describedby 連 errors
`<p className="text-xs text-status-failed">{errors[f.name]}</p>` 未透過 `aria-describedby` 與 `<Input>` 連結，screen reader 讀不到錯誤訊息。
**建議**：列入 D62 axe-core 規範範圍。

### G12 — Low — writeOverlay GC 僅在 `add`/`list` 觸發
若使用者開頁但不操作，過期 entries 不會主動清除（只在下次 add 時清）。記憶體洩漏雖小但語意不正確（「list 應反映 TTL 後消失」需重新觸發）。
**建議**：加 `setInterval` GC 或 hook 進 router transition。

### G13 — Low — `auditEvents.unshift(...)` 對 audit chain 順序未保證
v4 `auditImmutability.ts` 規範 audit chain hash 連續；mock unshift 只動 array，不算 hash。未來上 BE 會 chain 斷裂。
**建議**：mock 加 placeholder `prevHash` 欄位，留好 hook 點。

### G14 — Low — `withOverlay` 把 overlay items 排在 base 前，與 list 預設 `updatedAt desc` 排序衝突
overlay 雖然 `updatedAt = nowIso()`，但 `withOverlay` 直接 prepend，未走 sort，視覺上一致但語意上跳過排序層。
**建議**：`withOverlay` 改 merge 後依現行排序鍵重排。
